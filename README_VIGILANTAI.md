# 🚨 VigilantAI - Multi-Agent Adverse Event Reporting System

**Autonomous, end-to-end adverse event reporting using Anthropic Computer Use**

## 🎯 What It Does

VigilantAI automates the entire process of adverse event reporting from clinical data to FDA form submission:

1. **Strips all PHI locally** (HIPAA-compliant)
2. **Extracts adverse event data** with Claude
3. **Autonomously fills out government forms** using Computer Use

---

## 🏗️ Architecture

```

           ▼
┌──────────────────────────────────────────────────────┐

           │
           ▼
┌──────────────────────────────────────────────────────┐
│ AGENT 2: ADVERSE EVENT EXTRACTOR (Claude)           │

│ Output: fda_extracted_data.json                     │
└──────────┬───────────────────────────────────────────┘
           │
           ▼
┌──────────────────────────────────────────────────────┐
│ AGENT 3: COMPUTER USE FORM FILLER                    │
│ • Generates prompt for Anthropic Computer Use       │
│ • Maps FDA data to form fields                      │
│ • Handles multi-page navigation                     │
│ Output: computer_use_prompt.txt                     │
└──────────┬───────────────────────────────────────────┘
           │
           ▼
┌──────────────────────────────────────────────────────┐
│ ANTHROPIC COMPUTER USE (localhost:8080)              │

│ • STOPS before final submission                     │

```

## ⚡ Quick Start

### Prerequisites

1. **Docker running** (for Computer Use)
2. **Anthropic API key** set as environment variable
3. **Python 3.9+**

### Setup

```bash
# 1. Set API key
export ANTHROPIC_API_KEY="sk-ant-your-key-here"

# 2. Start Computer Use demo
# (Run this in a separate terminal)
docker run -d \
    -e ANTHROPIC_API_KEY=$ANTHROPIC_API_KEY \
    -v $(pwd):/host \
    -p 5900:5900 \
    -p 8501:8501 \
    -p 6080:6080 \
    -p 8080:8080 \
    ghcr.io/anthropics/anthropic-quickstarts:computer-use-demo-latest

# 3. Install Python dependencies (if needed)
pip install anthropic

# 4. Run the complete pipeline
python vigilantai_orchestrator.py
```

### What Happens

The orchestrator will:
1. ✅ Load `patient_adverse_event_case.json`
2. ✅ Run Agent 1 → Creates `patient_case_redacted.json`
3. ✅ Run Agent 2 → Creates `fda_extracted_data.json`
4. ✅ Run Agent 3 → Creates `computer_use_prompt.txt`
5. ✅ Give you instructions to complete Computer Use step

## 🎯 The Patient Case

The demo uses a realistic adverse event case:

**Patient:** 32-year-old female with atopic dermatitis

**Timeline:**
- Visit 1-2: Topical steroids (moderate improvement)
- Visit 3: Started Dupixent (dupilumab) - excellent eczema improvement
- Visit 4: Continued Dupixent - eczema well-controlled, mild dry eyes
- Visit 5: **SEVERE CONJUNCTIVITIS** → Dupixent discontinued

**Adverse Event:**
- Serious (required drug discontinuation, ophthalmology referral)
- Vision changes
- Probable drug-related (known Dupixent side effect)
- Reportable to FDA

## 🛡️ Safety Features

### 1. Local PHI Removal
- ALL de-identification happens locally
- NO patient data sent to Anthropic APIs before redaction
- Consistent tokenization (same patient = same token)

### 2. Computer Use Safety Gate
- Agent fills out form completely
- **STOPS before clicking Submit**
- Human review required
- Screenshot of completed form for verification

### 3. Multi-Layer Validation
- Agent 2 validates extracted data against source
- Date format conversion (DD-MMM-YYYY)
- Field mapping validation
